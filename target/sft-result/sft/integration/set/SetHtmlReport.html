Set html report

The DefaultConfiguration provide an HtmlReport to write test result in HTML.

The target of the HtmlReport resourcePath, will be copied to provide facilities files.
This resourcePath is a directory that contains all image, css and js files needed for the report generation and display.
For example, all css and js file of this directory will be included in the html head close.

Default report

The default configuration provide an html report

Change css and js

The most convenient way to custom your html report, is changing resourcePath.
You can modify native css and js file provide by SimpleFunctionalTest (copied in ./target/sft-result/sft-html-default/) in an local directory and then set its location as the resourcePath.

Css class used by HtmlReport are:
  • ignored
  • failed
  • succeeded
  • useCase
  • useCaseName
  • beforeUseCase
  • afterUseCase
  • comment
  • scenario
  • scenarioName
  • beforeScenario
  • afterScenario
  • instruction
  • exception
  • relatedUseCase
When html report use the default embedded resource sft-html-default
Then html report include path to js files ../../../sft-html-default/bootstrap.min.js, ../../../sft-html-default/jquery-1.10.2.min.js
And include path to css files ../../../sft-html-default/bootstrap-theme.min.css, ../../../sft-html-default/bootstrap.min.css, ../../../sft-html-default/sft.css
By setting the resource path to sut-html-resources
Then html report include path to js files ../../../sut-html-resources/override.js
And include path to css files ../../../sut-html-resources/override.css
