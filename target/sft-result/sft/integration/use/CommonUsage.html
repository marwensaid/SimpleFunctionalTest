Common usage: writing Use Cases

To become a use case, your class needs to use the junit annotation @RunWith(SimpleFunctionalTest.class)
Then all public methods of this class annotated with the junit   @Test are considered as scenario.
And all private or protected methods of this class are considered as fixture methods.
Class, methods are humanized in use case, scenario and fixture name.
Use case has cross/check/interrogation mark depending of test issue failed/succeeded/ignored.
Scenario has pink/green/yellow background with cross/check/interrogation mark depending of test issue failed/succeeded/ignored.
  • Succeeded fixture call are displayed with green check mark:    
  • Failed fixture call are displayed with red cross mark:    
  • Ignored fixture are displayed with yellow interrogation mark:    

Your first functional test

Create a simple JUnit test class and add JUnit annotation @RunWith(SimpleFunctionalTest.class)
When invoking JUnit
Then the fully qualified class name is converted into pah to html file
The functional test result is display in html
Class name is humanized as use case title
Test method is humanized as scenario section
Inner method call is humanized as fixture call
Successful use case scenario and fixture call are displayed with ending green check mark
Successful scenario section are also display with green background

When functional test failed

With a JUnit test class expecting failure
When invoking JUnit
Then the html file present failures
Failed use case and scenario are displayed with ending red cross mark
Successful fixture calls are displayed with green check mark. Until the failed fixture call displayed with red cross mark. Then the others ignored fixture calls are displayed with yellow interrogation mark.
Failed scenario section are also display with red background and with error stack trace
